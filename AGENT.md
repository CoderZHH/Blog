# agent.md

你是一个前端网页生成 Agent。请根据以下页面布局说明，生成一个个人博客网站首页。不要擅自重构整体布局，优先严格遵守页面分区、层级、滚动方式和视觉节奏。当前阶段先完成首页结构与基础交互，后续我会逐步调整细节。

---

## 1. 项目目标

生成一个具有个人风格的博客网站首页，整体是单页滚动布局，带有宇宙氛围、3D宇航员视觉核心、作品展示区、横向滚动卡片区、地图照片区、兴趣区和联系方式区。

页面重点不是传统博客列表，而是一个“个人展示 + 博客入口 + 作品集”的首页。

---

## 2. 整体布局要求

整个页面是移动端纵向长页面结构，但也要兼容桌面端显示。

整体从上到下分为以下区域：

1. 顶部菜单栏
2. Hero 首屏区域
3. 作品集大卡片区域
4. 横向滚动卡片区域
5. 地图 + 照片双栏区域
6. Hobby 区域
7. Contact 区域



---

## 3. 顶部菜单栏

页面最顶部有一个横向菜单栏。

要求：
- 菜单栏是一个独立圆角长条容器
- 宽度接近页面内容宽度
- 风格简洁
- 当前阶段菜单内容可以先用占位文本，例如：
  - Home
  - Works
  - Notes
  - Hobby
  - Contact

菜单栏整体不要复杂，不要做成传统重导航站点，更像一个轻量导航入口。

---

## 4. Hero 首屏区域

这是首页第一视觉重点区域。

### 4.1 区域结构
- 左侧：falling 宇航员 3D 展示区域
- 右侧：宇宙背景视觉区域

### 4.2 动效要求
这个区域需要有“宇航员和宇宙背景做微滚动视差”的感觉。

要求：
- 宇航员区域和背景区域不是静止的
- 页面滚动时，两者有轻微速度差
- 动效要克制，不要夸张
- 保持高级感和呼吸感

### 4.3 宇航员区域要求
- 宇航员是视觉主角
- 有“下落 / 失重 / 漂浮”感
- 默认是首页视觉主元素
- 后续这个位置可能接入真实 3D 模型，因此当前代码结构必须预留 3D 容器
- 宇航员区域单独成块，不能和背景混为一体

### 4.4 宇宙背景区域要求
- 作为 Hero 的背景主视觉部分
- 风格偏宇宙、深空、星云、粒子
- 背景不要受限于盒子里，直接铺满整个背景

---

## 5. 作品集区域

Hero 下方是一个大卡片区域，内容是“作品集”。

要求：
- 这是一个独立大模块
- 用于展示精选项目或作品入口
- 当前阶段可以只放一个标题 “作品集”

视觉要求：

- 与 Hero 区域保持统一视觉语言

## 6. 横向滚动卡片区域

这是页面中段的重要交互区域。

### 6.1 基本形式
一个横向滚动区域，里面排列多个小卡片。

当前阶段每个卡片先使用占位内容，例如：
- Obsidian new node
- Obsidian new node
- Obsidian new node
- Obsidian new node
- Obsidian new node

### 6.2 交互要求
- 该区域必须是横向滚动逻辑
- 在移动端允许自然横向滑动
- 在桌面端可以支持鼠标滚轮映射横向移动，或拖拽滚动
- 卡片之间有间距
- 每个卡片都是独立圆角模块

### 6.3 语义定位
这个区域可理解为：
- 博客笔记流
- 灵感节点
- 知识卡片
- 最近文章入口

但当前阶段先保持结构，不要过度填充内容。

## 7. 地图 + 照片区域

这个区域是双栏布局：

- 左边：Map

### 7.1 Map 区域

- 标注为 “Map (Like Apple Album)”

- 当前阶段可先使用占位区块，不必接真实地图 API
- 要保留后续替换成真实地图模块的空间

### 7.2 Photo 区域
要求：
- 右侧是一个较小卡片
- 标注为 “Photo”
- 适合作为照片展示入口
- 当前阶段可以先用占位图容器

### 7.3 布局要求
- 左右并排
- 左大右小
- 两个模块高度一致
- 与全站卡片圆角风格统一

- elegant motion

---

## 11. 动效原则

页面允许动效，但必须克制。

优先实现以下动效：

1. Hero 区域轻微视差
2. 横向滚动卡片区的自然滚动
3. 卡片 hover 时轻微浮起或阴影变化
4. 页面模块出现时有轻微 reveal 动画

禁止：
- 过度 bounce
- 夸张缩放
- 花里胡哨的粒子爆炸
- 大量无意义自动动画

目标是“有生命感”，不是“炫技”。

---

## 12. 响应式要求

必须支持响应式。

### 移动端
- 按手稿优先还原
- 所有模块上下排列
- Hero 保持大模块感
- 横向卡片区支持滑动

### 桌面端
- 在保留模块结构前提下适当拉宽
- Hero 可继续左右分区
- 地图和照片区域保持左右布局
- 页面整体不要被拉得太散

## 13. 技术实现要求

- 组件化拆分清晰

- 样式建议使用 Tailwind CSS
- 动效建议保持轻量，优先 CSS / Framer Motion

建议组件结构如下：

- `HeroSection`

- `ContactSection`
